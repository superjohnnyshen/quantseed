# QuantSeed SOP - 个人量化策略标准化投研流程

本 Skill 为**个人量化投资者**提供从策略立项到实盘上线的完整标准化流程，适用于任何技术栈（Python、QMT、自研框架均可）。当用户请求开发量化策略时，你必须严格按本 SOP 的 8 个阶段执行，**不可跳过任何阶段，不可提前查看样本外数据，不可反复调参**。

> 这是一个通用 Skill，可同时在 TRAE IDE（CodeBuddy）和 TRAE Work（WorkBuddy）中使用。

---

## 何时调用此 Skill

在以下场景中**必须**调用此 Skill：

- 用户要求开发新的 A 股量化策略
- 用户要求评估、回测或调试已有策略
- 用户提到"选股"、"因子"、"回测"、"实盘"、"量化"等关键词
- 用户要求修改策略参数或新增特征
- 用户要求上线策略到模拟盘或实盘

**不适用场景**：高频/日内交易、加密货币、纯基本面/事件驱动策略。

## Failure Handling

当遇到以下情况时，你必须按对应策略处理，不得自行发挥：

| 失败场景 | 处理方式 |
|---------|---------|
| 用户要求跳过某个阶段 | 拒绝并解释该阶段的必要性，引用 SOP 案例说明跳过的后果 |
| 用户要求查看样本外数据 | 拒绝，重申"样本外封存至第六阶段盲测"原则 |
| 用户反复调参超过 2 次 | 拒绝，引用参数冻结原则和仓位加权案例 |
| 用户想用前复权（qfq）数据 | 拒绝，引用复权口径案例，要求改用后复权（hfq） |
| 用户策略类型无法判断 | 询问用户策略类型，确认后选择轻量或完整路径 |
| 用户偏离当前阶段讨论后续阶段 | 提醒用户当前阶段未完成，建议先完成当前交付物 |
| 用户要求上线但未经过盲测 | 拒绝，引用红线 #10 |

## 两条路径（根据用户策略类型选择）

根据用户策略的复杂度，选择不同路径。**不要对所有策略都走完整 8 阶段——新手用规则型策略验证想法时，走轻量路径即可。**

| 路径 | 适用场景 | 走哪些阶段 |
|------|---------|-----------|
| **轻量路径** | 规则型策略（均线、动量、纯排序），学习/验证想法，不涉及 ML 模型 | ①立项 → ②数据 → ④回测 → ⑥盲测 |
| **完整路径** | ML 模型策略（LGBM 等），准备上实盘 | 全部 8 个阶段 |

**关键判断**：如果用户策略中没有 ML 模型训练环节，直接走轻量路径，跳过第三阶段（因子工程）和第五阶段（风控调参）。

## 数据源选择指南

个人量化常用的三种数据源。你必须根据用户的场景推荐合适的数据源。

| 数据源 | 适用场景 | 优点 | 缺点 | 成本 |
|--------|---------|------|------|------|
| **AkShare** | 学习/入门、验证想法、轻量路径 | 完全免费、API 简单、无需注册 | 数据质量一般、偶发不稳定、部分字段缺失 | 0 元 |
| **Tushare** | 进阶开发、回测、完整路径 | 数据质量高、稳定、字段完整 | 需要积分（捐赠 200 元起）、有调用频率限制 | 200 元起 |
| **QMT SQLite** | 实盘对齐、上线运行 | 和实盘完全一致、无网络依赖 | 只能在安装了 QMT 的电脑上使用、数据仅限本地 | 开户费用 |

### 选择建议

- **新手入门**：优先推荐 **AkShare**，零门槛，先跑通策略再说
- **回测验证**：建议使用 **Tushare**，数据质量高，回测结果可靠
- **实盘上线**：必须使用 **QMT SQLite**，确保回测/模拟盘/实盘数据完全一致
- **进阶用户**：可以同时配置多个数据源，用 Tushare 做回测，用 QMT 做实盘验证

## 核心原则（不可违反）

| 原则 | 说明 |
|------|------|
| 流程不可逆 | 严格按阶段顺序，上阶段验收不通过，禁止进入下阶段 |
| 数据隔离 | 训练集/验证集/样本外集物理隔离，样本外封存至盲测阶段 |
| 参数冻结 | 所有参数仅在训练集内单次调整，完成即永久冻结 |
| 规则对齐 | 回测规则必须 100% 匹配实盘操作 |
| 全程留痕 | 所有操作、调参、回测结果书面记录，版本化管理 |
| 回测前自动校验 | 每次回测前必须通过 look-ahead/数据对齐/特征完整性三项检查 |
| 消融必做 | 任何新增特征/过滤/参数，必须做消融测试验证独立贡献 |
| 数据快照 | 每个大版本的数据集冻结快照，禁止回测期间增量更新 |

## 八阶段流程

### 第一阶段：策略立项与目标定义

**目的**：防止无目标开发，杜绝事后根据回测改标准。

你必须帮助用户明确以下内容，生成《策略立项单》：

- 策略名称（唯一标识）
- 交易逻辑（一句话描述核心逻辑）
- 交易周期（日内/隔夜/周度/月度）
- 持仓时长（最长持有天数）
- 资产范围（股票池范围）
- 硬性指标：目标年化、目标夏普、最大回撤红线、单年最大亏损
- 信号区分度门槛：Bin9 胜率 ≥ 52% 或 RankIC ≥ 0.02
- 仓位限制：单票仓位上限、总仓位上限
- 手续费：佣金万1（最低5元）、印花税万5（仅卖出）、滑点分三档

**数据集划分**（完成后封存样本外）：

| 数据集 | 用途 |
|--------|------|
| 训练集 | 模型训练 + 参数调优 |
| 验证集 | 超参选择（Walk Forward 滚动） |
| 样本外集 | 全阶段封存，仅第六阶段盲测使用 |

**禁止**：提前查看样本外集；模糊定义目标事后修改。

---

### 第二阶段：数据基建与校验

**目的**：建立统一、干净、可追溯的数据流水线。

**必须执行**：
1. 选定数据源，固定接口与字段名
2. 固定复权口径与交易日历，全程不可变更
3. 自动化清洗：剔除 ST/*ST/退市/长期停牌；标记涨跌停封死/停牌；处理异常值与跳空
4. 列式存储（建议 Feather/Parquet），按需分区
5. 编写每日增量更新脚本

#### 数据质量标准（七项必检）

**1. 幸存者偏差（致命）**
- 退市股历史数据必须保留，退市日之后才标记不可交易
- ST 标记须按时间序列存储，不可用当前快照覆盖历史
- 持仓退市日必须强制平仓

**2. 复权口径（严重）**
- 必须使用后复权（hfq），前复权（qfq）导致历史价格漂移
- 存储 `adj_factor` 列，可随时转换
- 禁止混用复权口径

**3. 停牌处理（严重）**
- 停牌日标记 `is_suspended`
- `.shift(N)` 须用交易日计数而非自然日
- 停牌期间持仓按最后收盘价估值，不得冻结在买入成本

**4. 费率标准（致命）**
- 佣金：`max(amount × 0.0001, 5.0)`，不可忽略最低5元
- 印花税：仅卖出，万5（0.05%）
- 滑点分层：成交额 >5亿/万三、>5000万/万五、<5000万/千一

**5. 模拟盘与回测对齐（严重）**
- 仓位计算公式必须逐项一致
- 必须包含 `stop_mult` 因子

**6. 异常值与缺失值**
- 涨跌幅缩尾：主板 ±9.5%、科创板 ±19.5%
- PE/PB：99分位以上截断，负值置 NaN，前向填充

**7. 数据源一致性**
- 全项目统一使用一个数据源，禁止混用
- 禁止 API 代理

### 第三阶段：因子工程与模型研发

**单因子检验标准**：

| 检验项 | 方法 | 通过标准 |
|--------|------|----------|
| 分组单调性 | 全训练集分10 Bin | 单调递增/递减 |
| IC 均值 | 日频 IC 序列均值 | ≥ 0.02 |
| ICIR | IC 均值/IC 标准差 | ≥ 0.3 |
| 分年度稳定性 | 每年独立计算 IC | 各年方向一致 |
| 极端行情表现 | 熊市/震荡市单独检验 | 不出现大幅反向 |

**剔除规则**：IC 长期为负、分层杂乱、年度波动剧烈的因子直接丢弃。**直觉是因子工程的敌人，IC 才是唯一的裁判。**

**模型训练标准**：
1. 目标标签与策略交易逻辑一致
2. 严格时间序列分割——使用 Walk Forward 滚动窗口，禁止随机 shuffle
3. 超参搜索轮次 ≤ 200
4. 优先看稳健性指标（IC 均值、ICIR、Bin 单调性），而非单区间收益
5. 随机打乱标签对照测试，确认模型不是随机拟合

**禁止**：用样本外数据训练；无限制扩大超参搜索空间；为提高收益随意堆因子。

---

### 第四阶段：回测引擎开发与交易规则对齐

**引擎必须包含的模块**：

| 模块 | 必须实现 |
|------|----------|
| 选股 | 严格按规定时点可用的数据生成信号 |
| 买入撮合 | 买入时点、价格逻辑与实盘一致 |
| 卖出撮合 | 卖出触发条件、成交价格与实盘一致 |
| 不可交易 | 涨跌停/停牌当日跳过，顺延逻辑明确 |
| 仓位管理 | 单票上限、总仓位上限、现金/债券管理 |
| 交易成本 | 佣金、印花税、滑点（分档）全部写入 |

**回测双模式**：
- 快速模式：仅用于淘汰垃圾模型，不可用于上线决策
- 实盘对齐模式：策略评估/参数校验/WF/样本外，唯一可用

#### 回测前自动校验（每次回测前必须通过）

1. **前视偏差**：所有特征列不存在 `shift(-N)` 或 `iloc[i+N]`
2. **数据对齐**：标签（fwd_ret）与当日（change_pct）相关系数 < 0.15
3. **特征完整性**：训练集特征非空率 > 80%
4. **标签分布**：正负样本比在 0.3~0.7 之间
5. **时间窗口**：训练集最大日期 < 样本外最小日期

**禁止**：回测规则与实盘不一致；忽略交易成本与滑点；使用未来函数。

---

### 第五阶段：风控参数调优与过拟合防控

**硬性规则**：
1. 仅允许在训练集内调参
2. 同类型参数仅允许单次调整，完成即永久冻结
3. 候选参数数量 ≤ 2 组
4. 禁止为抹平某几年历史收益针对性修改参数
5. 所有候选参数和选定依据必须书面记录

#### 消融测试（必做）

任何新增特征/过滤/参数，必须做消融测试：
```
基线回测（无该项） → 加入该项 → 对比 OOS 年化/夏普/回撤/胜率
Δ年化 > +2pp 或 Δ夏普 > +0.1 → 保留
否则 → 删除
```

**禁止**：使用样本外数据调参；反复迭代同一参数；网格遍历风控参数。

---

### 第六阶段：样本外盲测与全维度验收

**盲测流程**：
1. 锁定全部参数（模型+风控+交易规则）
2. 一次性运行封存的样本外数据
3. 仅查看三大全局指标：年化收益、夏普、最大回撤
4. **禁止拆分查看单年收益**
5. 对比《策略立项单》硬性目标，逐项核验

**全维度验收清单**：
- 未来函数排查
- 净度曲线平滑性（无极端回撤）
- 结构性 alpha 底牌（随机信号替代模型，确认基础规则不产生伪 alpha）
- 极端行情压力测试（熊市、深度调整、震荡年份）
- 交易逻辑抽查（随机抽取 N 笔，人工核对）
- 数据集完整性（训练/样本外无交叉污染）

**验收结论**：
- 全部达标 → 通过，进入模拟盘
- 核心指标不达标 → 参数回退至上一版本，放弃本次修改

**禁止**：看完样本外结果后回头修改任何参数；拆分查看样本外单年收益。

---

### 第七阶段：上线分层试运行

**三阶段推进**：

| 阶段 | 时长 | 资金比例 | 任务 |
|------|------|----------|------|
| 模拟盘 | ≥ 2 周 | 0% | 验证链路零报错 |
| 小资金实盘 | ≥ 1 个月 | ≤ 10% | 对比回测/模拟盘/实盘绩效 |
| 梯度扩容 | 每次 ≥ 1 个月 | 30%→60%→100% | 监控容量与滑点变化 |

**模拟盘必须验证**：
- 买/卖撮合逻辑与回测一致
- 数据及时性
- 异常处理（涨跌停、停牌、数据缺失）
- 净值计算准确

**禁止**：实盘规则与回测不一致；一次性满仓投入。

#### 实盘心理纪律

当用户策略进入实盘阶段时，你必须提醒以下行为纪律：

1. **连续盈利不放大仓位** — 仓位只按策略规则执行，不因近期盈亏手动调整
2. **连续亏损不恐慌砍仓** — 只要策略没触发风控红线，就按信号执行
3. **盯盘不动手** — 盘中任何操作都是冲动交易，不在策略框架内
4. **亏了不换策略，赚了不迷信策略** — 策略需要至少一个季度才能评价
5. **每周只看一次净值** — 每天看净值只会制造焦虑，导致频繁干预

---

### 第八阶段：版本管理与迭代规范

**版本编号**：
- 大版本（V2.0）：新增因子/换模型架构 → 必须重新走完整 SOP
- 小版本（V2.1）：仅修改风控参数/交易成本 → 第五阶段起重新验证

**每个版本必须包含**：策略代码（git tag）、模型文件、参数配置、回测报告、验收报告。

**版本回退**：
1. 立即回退到上一个稳定版本
2. 记录回退原因和时间
3. 修复后重新走第五阶段起验证
4. 禁止"边跑边改"

#### 数据快照版本化

每个大版本必须创建独立只读数据快照：
- 导出 hfq 日线全量 → `V{N}_{date}_prices.parquet`
- 导出 PE/PB/基本面全量 → `V{N}_{date}_basic.parquet`
- 记录数据截止日期 → `V{N}_{date}_meta.json`
- 后续回测/模拟盘统一从快照读取

## 过拟合防控 10 条红线

在帮助用户开发策略时，你必须主动检查并阻止以下行为：

| # | 红线 |
|---|------|
| 1 | 禁止使用样本外数据训练模型、调参或优化规则 |
| 2 | 禁止反复迭代修改同一参数；同类型参数仅允许单次调整 |
| 3 | 禁止大规模网格遍历参数；候选参数 ≤ 2 组 |
| 4 | 禁止为抹平特定年份历史收益而针对性修改参数 |
| 5 | 禁止拆分查看样本外单年收益；仅允许查看全局指标 |
| 6 | 禁止回测规则与实盘操作不一致 |
| 7 | 禁止忽略交易成本、滑点、涨跌停等现实约束 |
| 8 | 禁止使用未来函数；信号计算不得依赖未来数据 |
| 9 | 禁止追求完美回测曲线；接受策略天然的年度收益分化 |
| 10 | 未经样本外盲测验收，禁止上线实盘 |

## 关键失败案例（必须引以为戒）

当用户出现以下倾向时，引用对应案例警示：

1. **"这个因子听起来有道理，加进去试试"** → 引用 moneyflow 因子案例：RankIC = -0.017，特征重要性倒数第一。直觉是敌人，IC 是裁判。

2. **"再调一次参数，训练集还能更好"** → 引用仓位加权案例：夏普 +0.08 但回撤 +2.8%，不是 alpha 是放大器。

3. **"多加两个特征应该能提升"** → 引用 V5 特征扩展案例：AUC 仅 +0.0089，不值得增加复杂度。

4. **"换个数据源试试"** → 引用复权口径案例：Tushare qfq 回测夏普 1.8，QMT 模拟盘三天亏 2%，排查两天才定位。

5. **"跳过模拟盘直接上小资金"** → 引用模拟盘案例：模拟盘第一周就暴露了 `stop_mult` 缺失和佣金最低5元两个 bug，跳过的代价是真金白银。
